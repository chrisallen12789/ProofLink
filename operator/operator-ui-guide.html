Principle 1 Lead with the next move, not raw data density

Each workspace should open with a calm summary and a short list of actions that move work forward immediately.

Principle 2 Keep live field pressure visible across the record

Dispatch gaps, load pressure, receivables, and site memory should stay visible without forcing a tab switch.

Principle 3 Use the same shell everywhere

Jobs, dispatch, compliance, and money should all reuse the same workspace shell so the product feels deliberate instead of stitched together.
