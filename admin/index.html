SQL migration files

Run in your Supabase SQL editor in order:

  1. sql/catchup_run_this.sql - Core live schema for tenants, operators, storefront, customers, orders, payments, and RLS
  2. sql/service_workflow_phase1.sql - Adds leads, bids, jobs, payment-state normalization, and service workflow RPCs

Before hosted integration or e2e service-workflow tests, run npm run test:preflight:service-workflow against that same project.

Grant admin access

Any operator can currently use this admin panel. To restrict to admin-only once roles are set up: 

UPDATE operators SET role = 'admin' WHERE email = 'you@example.com';

Then swap requireOperatorContext → requireAdminContext in admin functions.
